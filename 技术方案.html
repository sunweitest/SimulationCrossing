产品简介

玩家选择一个小说/历史时代，扮演预设或自定义角色，通过自然语言与 AI 互动， 推动剧情发展。后端通过 DashScope（阿里云百炼平台）调用通义千问大模型生成场景、 选项和成就奖励，前端使用 Vue 3 构建 SPA。

技术架构

层技术说明
前端 Vue 3 + Vite 5 + Pinia + Vue Router + Axios SPA，开发端口 5173，Vite 代理 /api 到后端
后端 FastAPI 0.109 + SQLAlchemy 2.0 + Alembic + Pydantic v2 异步 REST API，端口 8000
数据库 PostgreSQL 15 + asyncpg 数据库名 simulation_crossing，表结构由 Alembic 管理
缓存/限流 Redis 7 每日消息次数限制（免费 26 条，登录 +10）
AI 引擎 DeepSeek (OpenAI 兼容) + DashScope (通义千问/embedding) 双 Provider 模式，LLM_PROVIDER 配置切换；text-embedding-v4 做场景图片匹配

数据库迁移

使用 Alembic 管理所有表结构变更。迁移脚本位于 backend/alembic/versions/。

执行迁移：

cd backend
E:\pythonProject\InteractiveNovel\novel\Scripts\python.exe -m alembic upgrade head

生成新迁移：

E:\pythonProject\InteractiveNovel\novel\Scripts\python.exe -m alembic revision --autogenerate -m "描述"

⚠ 注意：AUTO_CREATE_TABLES=false，不要在启动时自动建表，始终通过 Alembic 管理。

关键设计决策

  • 异步全链路：FastAPI + SQLAlchemy Async + asyncpg，避免 IO 阻塞。
  • Alembic 管理表结构：AUTO_CREATE_TABLES=false，不在启动时隐式建表，防止生产环境意外变更。
  • 未登录可用：游客也能创建会话和游玩，但每日消息次数更少（26 vs 36），且数据不绑定账号。
  • Redis 限流降级：Redis 不可用时自动放行，不阻断游戏体验。
  • LLM 返回解析容错：JSON 提取失败时使用 fallback 场景，确保游戏不中断。
  • 角色-时间节点约束：预设角色在 timeline_scenes 中定义合法时间节点，API 层强制校验。
  • 场景图片向量匹配：启动时用 text-embedding-v4 将图片文件名（中文描述）转为 1024 维向量缓存到磁盘。运行时嵌入剧情文本 → 余弦相似度匹配最佳图，作为全屏背景。文件指纹（SHA-256）自动检测增删改并重建缓存。
