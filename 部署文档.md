
## 4. 配置 PostgreSQL 数据库

```bash
# 切换到 postgres 用户
sudo -u postgres psql
```

在 psql 中执行：

```sql
-- 创建数据库用户（替换 your_password 为强密码）
CREATE USER novel_user WITH PASSWORD 'your_password_here';

-- 创建数据库
CREATE DATABASE simulation_crossing OWNER novel_user;

-- 授予权限
GRANT ALL PRIVILEGES ON DATABASE simulation_crossing TO novel_user;

-- 退出
\q
```

验证连接：

```bash
psql -U novel_user -d simulation_crossing -h localhost -W
# 输入密码后应能成功连接
```

## 5. 配置 Redis

Ubuntu 默认安装后 Redis 仅监听 127.0.0.1，无需额外配置即可使用。

```bash
# 确认 Redis 运行中
sudo systemctl status redis-server

# 可选：设置 Redis 密码（生产环境建议）
sudo nano /etc/redis/redis.conf
# 找到 # requirepass foobared 这一行，取消注释并修改密码：
# requirepass your_redis_password
# 保存后重启：
sudo systemctl restart redis-server
```

> **注意**：如果设置了 Redis 密码，需要在 `.env` 中配置 `REDIS_PASSWORD`（或修改项目 config.py 以支持）。

## 9. 前端部署

### 方案选择

| 方案 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| **A — 服务器构建** | 上传源码 → 服务器 `npm run build` | 一劳永逸，源码和产物一致 | 服务器需装 Node.js |
| **B — 本地构建上传** | 本地 `npm run build` → 只传 `dist/` | 服务器不需要 Node.js | 每次更新都要手动传 |

> **推荐**：首次部署用方案 A（服务器构建）。日常小更新用方案 B（本地构建传 dist）。

### 方案 A：服务器构建（首次部署推荐）

```bash
cd /opt/novel/frontend

# 安装依赖
npm install

# 构建生产版本（输出到 dist/ 目录）
npm run build
```

构建产物在 `/opt/novel/frontend/dist/`，Nginx 直接指向这个目录。

### 方案 B：本地构建再上传（日常更新推荐）

在本地 Windows 机器上：

```bash
# 1. 本地构建
cd E:\pythonProject\InteractiveNovel\SimulationCrossing\frontend
npm run build
```

构建完成后，目录结构：
```
frontend/dist/
├── index.html
├── assets/          # CSS、JS 等
└── ...
```

```bash
# 2. 上传 dist 到服务器
scp -r dist/* ubuntu@你的服务器IP:/opt/novel/frontend/dist/

# 或者 rsync（增量同步，更快）
rsync -avz dist/ ubuntu@你的服务器IP:/opt/novel/frontend/dist/
```

> **注意**：`dist/` 目录不需要进 git，`.gitignore` 已包含。每次 `npm run build` 会清空重建。

## 10. 使用 Nginx 反向代理

### 10.1 创建 Nginx 配置

```bash
sudo nano /etc/nginx/sites-available/novel
```

写入以下内容：

```nginx
server {
    listen 80;
    server_name 你的域名或IP;

    # 前端静态文件
    root /opt/novel/frontend/dist;
    index index.html;

    # 日志
    access_log /var/log/nginx/novel_access.log;
    error_log  /var/log/nginx/novel_error.log;

    # 前端 SPA 路由（可选，如果需要 Vue Router history 模式）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 场景图片（public 目录下）
    location /images/ {
        alias /opt/novel/frontend/public/images/;
    }

    # 后端 API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Connection '';

        # SSE 流式传输支持（重要！）
        proxy_buffering off;
        proxy_cache off;
        gzip off;
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;

        # 上传文件大小限制（批量导入 xlsx）
        client_max_body_size 10M;
    }

    # 可选：限制访问管理后台
    location /admin {
        try_files $uri /index.html;
    }
}
```

### 10.2 启用站点

```bash
# 创建软链接启用站点
sudo ln -s /etc/nginx/sites-available/novel /etc/nginx/sites-enabled/

# 删除默认站点（避免冲突）
sudo rm -f /etc/nginx/sites-enabled/default

# 检查配置语法
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx
```

### 10.3 配置 HTTPS（强烈推荐生产环境）

使用 Let's Encrypt 免费证书：

```bash
# 安装 certbot
sudo apt install -y certbot python3-certbot-nginx

# 自动获取证书并配置 Nginx
sudo certbot --nginx -d 你的域名
```

完成后，Certbot 会自动修改 Nginx 配置添加 HTTPS 和自动续期。

## 11. 使用 systemd 守护进程

### 11.1 创建后端服务

```bash
sudo nano /etc/systemd/system/novel-backend.service
```

内容：

```ini
[Unit]
Description=AI Interactive Novel - Backend
After=network.target postgresql.service redis-server.service
Wants=postgresql.service redis-server.service

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/opt/novel/SimulationCrossing/backend
Environment=PATH=/opt/novel/SimulationCrossing/backend/venv/bin:/opt/venv/bin:/usr/local/bin:/usr/bin:/bin
ExecStart=/opt/novel/SimulationCrossing/backend/venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=3
StandardOutput=journal
StandardError=journal

# 安全加固
PrivateTmp=true
NoNewPrivileges=true

[Install]
WantedBy=multi-user.target
```

> **说明**：
> 
> - `--host 127.0.0.1`：只监听本地，由 Nginx 反向代理对外提供服务
> - `--port 8000`：与前端 Nginx proxy_pass 一致
> - 不需要 `--reload`（生产环境不建议热重载）

### 11.2 设置文件权限

```bash
# 确保 www-data 用户可以访问项目文件
sudo chown -R www-data:www-data /opt/novel

# 日志目录需要可写
sudo mkdir -p /opt/novel/backend/logs
sudo chown -R www-data:www-data /opt/novel/backend/logs

# 缓存目录需要可写（场景图片 embedding 缓存）
sudo mkdir -p /opt/novel/backend/cache
sudo chown -R www-data:www-data /opt/novel/backend/cache
```

### 11.3 启动服务

```bash
sudo systemctl daemon-reload
sudo systemctl enable novel-backend
sudo systemctl start novel-backend

# 查看状态
sudo systemctl status novel-backend

# 查看日志
sudo journalctl -u novel-backend -f
```

### 12.1 基础健康检查

```bash
# 后端 API 健康检查
```

### 12.2 端到端测试

1. 浏览器访问 `http://你的服务器IP或域名`
2. 应看到角色创建页面（全屏沉浸式背景）
3. 小说下拉框应显示 5 个选项（从 API 动态加载）
4. 选择一个小说 → 时间节点下拉自动更新
5. 选择角色 → 点击"开始穿越"
6. 进入游戏 → 剧情应该流式逐字出现
7. 场景背景图应自动匹配（有 embedding 缓存时）
8. 访问 `/admin` → 输入管理密钥 → 进入管理后台
9. 测试小说 CRUD、角色批量导入模板下载

### 12.3 常见问题排查

| 问题              | 排查方法                                       |
| --------------- | ------------------------------------------ |
| 502 Bad Gateway | 后端是否启动：`systemctl status novel-backend`    |
| 数据库连接失败         | `.env` 中 `DATABASE_URL` 密码是否正确             |
| 角色创建页小说为空       | Redis 是否运行、`GET /api/game/novels` 是否 200   |
| 场景图片不显示         | `frontend/public/images/scene/` 是否有 png 文件 |
| SSE 流式中断（ERR_INCOMPLETE_CHUNKED_ENCODING） | ① Nginx 是否配置 `proxy_http_version 1.1` + `gzip off` + `proxy_buffering off`；② 后端日志检查 SQLAlchemy `UnboundLocalError` 或 `Instance is not persistent`；③ 确认 `email-validator` 已安装 |
| 管理后台 403        | `X-Admin-Key` 请求头是否正确传递                    |

## 附录 C：生产环境实测修复记录

> 以下为 2026-07-04 在 Ubuntu 24.04 服务器上实际部署时遇到的三个问题及修复。

### C.1 SSE 流式剧情中断（ERR_INCOMPLETE_CHUNKED_ENCODING）

**根因（三层问题）**：

| 层 | 问题 | 修复 |
|---|---|---|
| 后端 Python | `game_session` 在异步生成器 `event_stream()` 中被 SQLAlchemy detach，`Instance is not persistent within this Session` | 在生成器开头重新从数据库查询 `game_session`，全程使用局部变量 `game_session_local` |
| 后端 Python | 重查引入的变量作用域冲突 → `UnboundLocalError` | 统一使用 `game_session_local`，不在生成器内重新赋值同名的闭包变量 |
| Nginx | 默认 HTTP/1.0 代理不支持 chunked encoding；全局 `gzip on` 缓冲 SSE 流 | API location 加 `proxy_http_version 1.1` + `gzip off` + `proxy_set_header Connection ''` |

### C.2 缺失依赖

`requirements.txt` 未包含 `email-validator`，导致 pydantic EmailStr 校验启动失败：

```bash
pip install email-validator
```

### C.3 本机部署路径

| 配置项 | 值 |
|---|---|
| 项目根目录 | `/opt/novel/SimulationCrossing/` |
| 虚拟环境 | `/opt/venv/`（独立于项目目录） |
| PostgreSQL 用户 | `novel_user` |
| Redis 端口 | `6379`（无密码） |
| 公网 IP | `119.91.233.170` |
